# 🚀 TeamUp 프로젝트: MVP 기능 및 데이터 명세서

## 1. 핵심 기능 정의 (Core Features)

### ① 다이내믹 플레이어 카드 (FIFA Style Profile)

> "단순한 텍스트 프로필은 가라. 나는 '카드'로 말한다."
> 
- **개요:** 사용자의 신체 조건, 포지션, 실력, 스타일을 시각화된 **'카드(Card)'** 형태로 보여줍니다.
- **차별점:** 활동량과 받은 평가에 따라 카드 배경(Skin)과 칭호가 업그레이드되는 **성장형 프로필**입니다.
- **UI 컨셉:** FIFA FUT 카드, NBA 2K 선수 카드 스타일.

### ② NBA DNA 팀 빌딩 (Legendary Team HQ)

> "우리 팀은 시카고 불스의 정신을 계승한다."
> 
- **개요:** 팀 생성 시 **3가지 NBA 레전드 팀 스타일(불스/워리어스/스퍼스)** 중 하나를 선택합니다.
- **AI 페르소나:** 선택한 스타일에 따라 팀 전담 AI 매니저의 **말투, 조언 방식, 강조하는 전술**이 달라집니다.
- **성장 시스템:** 경기 판수에 따라 팀 레벨(Lv)이 오르는 RPG 요소 도입.

### ③ 직관적 지도 매칭 (Map-Based Matching)

> "복잡한 검색 없이, 지도 위에서 직거래한다."
> 
- **개요:** 내 주변 반경 5km 내의 '매치' 및 '용병' 모집 글을 핀(Pin)으로 확인합니다.
- **Flow:** 지도 핀 클릭 → 요약 정보 확인 → **[카카오톡 오픈채팅]** 버튼 클릭 (즉시 연결).

### ④ 원터치 AI 코칭 (One-Touch Feedback)

> "입력은 3초, 피드백은 평생."
> 
- **개요:** 경기 직후 승패와 '아쉬운 점(태그)' 하나만 선택하면 끝납니다.
- **AI 역할:** 입력된 데이터를 기반으로, 우리 팀 DNA(예: 불스 스타일)에 맞는 **격려와 전술 꿀팁**을 텍스트로 제공합니다. (특정인 비난 절대 불가)

## 2. 데이터 구조 명세 (Data Structure)

개발자가 DB 설계를 할 때 바로 참고할 수 있는 포맷입니다.

### 👤 User (사용자: 플레이어 카드 데이터)

피파 카드처럼 보여주기 위한 핵심 재료들입니다. `SkillLevel`은 화면엔 문자로, 내부적으론 점수로 처리합니다.

| **필드명 (Field)** | **타입 (Type)** | **상세 내용 (Description)** | **비고** |
| --- | --- | --- | --- |
| `nickname` | String | 닉네임 | 카드 상단 표시 |
| `region` | String | 주 활동 지역 (예: 마포구) | 매칭 기준 |
| `height` | Integer | 키 (cm) | 농구 필수 스탯 |
| `position` | Enum | **G / F / C** (가드/포워드/센터) | 카드 아이콘 |
| `sub_position` | Enum | G / F / C (부 포지션) | 선택 사항 |
| `play_style` | Enum | **SL**(돌파), **SH**(슈터), **DF**(수비), **PA**(패스) | **카드 스타일 결정** |
| `skill_level` | Enum | **ROOKIE**(10점) ~ **PRO**(90점) | **분석용 점수 매핑** |
| `card_skin` | String | `DEFAULT`, `GOLD`, `RARE` | 카드 디자인 등급 |
| `status_msg` | String | 한 줄 각오 (20자) | 카드 하단 문구 |

> 💡 Skill Level 분석 로직 (Enum)
> 
> - `ROOKIE` (입문) = 10점
> - `BEGINNER` (초보) = 30점
> - `INTERMEDIATE` (중수) = 50점
> - `ADVANCED` (고수) = 70점
> - `PRO` (선출) = 90점

### 🛡️ Team (팀: 구단 운영 데이터)

팀의 정체성(DNA)과 레벨을 관리합니다.

| **필드명 (Field)** | **타입 (Type)** | **상세 내용 (Description)** | **비고** |
| --- | --- | --- | --- |
| `name` | String | 팀 이름 | 유니크 |
| `leader_id` | Long | 팀장 ID | User PK |
| `team_dna` | Enum | **BULLS** (수비/투지), **WARRIORS** (3점/재미), **SPURS** (패스/기본) | **AI 성격 결정** |
| `team_level` | Integer | 팀 레벨 (1 ~ 99) | 판수 비례 상승 |
| `team_exp` | Integer | 현재 경험치 | 100XP마다 레벨업 |
| `emblem_url` | String | 팀 로고 이미지 | 기본/업로드 |

### 📍 Post (매칭 글: 지도 데이터)

지도에 뿌려질 핀 정보입니다.

| **필드명 (Field)** | **타입 (Type)** | **상세 내용 (Description)** | **비고** |
| --- | --- | --- | --- |
| `latitude` | Double | 위도 | 지도 좌표 |
| `longitude` | Double | 경도 | 지도 좌표 |
| `type` | Enum | **MATCH** (팀대결), **GUEST** (용병) | 핀 색상 구분 |
| `game_time` | DateTime | 경기 시작 시간 |  |
| `kakao_link` | String | 오픈채팅방 URL | 필수 입력 |

### 📝 Record (경기 기록: AI 피드백 재료)

AI가 코멘트를 생성하기 위해 필요한 최소한의 팩트입니다.

| **필드명 (Field)** | **타입 (Type)** | **상세 내용 (Description)** | **비고** |
| --- | --- | --- | --- |
| `team_id` | Long | 팀 ID |  |
| `result` | Enum | **WIN / LOSE** | 승패 |
| `feedback_tag` | Enum | **DEFENSE**(수비), **OFFENSE**(공격), **MENTAL**(멘탈) | AI 분석 주제 |
| `ai_comment` | Text | AI가 생성한 조언 | 히스토리 저장 |

### 💡 Tech Lead의 상세 분석 (Gap Check)

준호님께서 혹시 헷갈리실까 봐, **4번(팀 매칭)** 부분이 어떻게 구현되었는지 조금 더 자세히 설명해 드릴게요.

**Q. "팀 vs 팀 매칭 기능(4번)"이 왜 지도(3번)랑 AI코칭(4번)으로 쪼개졌나요?**

- **기존 생각:** "매칭" = "알고리즘이 상대를 찾아주는 것"
- **변경 기획:** "매칭" = **"지도에서 상대를 찾고(Find) + 경기 후 결과를 기록하는(Record) 전체 과정"**
- **구현:**
    - **지도(Map):** 여기서 `Post Type: MATCH` 핀을 꽂으면 그게 바로 **'팀 매칭 기능'**입니다. (준호님의 4번 요구사항 충족)
    - **AI 코칭:** 매칭만 하고 끝내면(당근마켓처럼) 우리 서비스의 핵심인 'AI'가 소외됩니다. 그래서 **"매칭된 경기의 결과"를 AI가 피드백해주는 기능**을 MVP의 마지막 퍼즐로 붙인 것입니다.

### 🏁 최종 결론

- 개인 프로필 ➡️ **FIFA 카드**
- 개인 참여 ➡️ **지도 용병 핀**
- 팀/AI 생성 ➡️ **NBA DNA 선택**
- 팀 매칭 ➡️ **지도 매치 핀 + 경기 후 AI 코칭**
